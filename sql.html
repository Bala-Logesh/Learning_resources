1. SQL Overview

1.1. Specifying the database to use (USE Statement)

We can specify which database to use at a point of time by the help of USE statement viz 

USE database_name;

1.2. The SELECT Statement

It is used to retrieve the contents (columns) or it can also be used on a plain string or number. If it is used in conjunction with the AS clause, then the AS clause (re)names the column.

The ORDER BY clause is used to order a column from the columns retrieved via the SELECT clause. Example syntax is shown below 

SELECT * FROM Country ORDER BY Name;

Specific columns can be retrieved by specifying the column names next to the SELECT clause and the column names being delimited by ,. A specific column name can be given an alias by using the AS clause viz 

SELECT Name, LifeExpectancy AS "Life Expectancy" from Country;

The difference between the use of '' and "" is that '' is used for a literal string and most other things like identifiers use "".

The following still works (i.e. even in the elimination of AS clause) but it is not a good practice. 

SELECT Name, LifeExpectancy "Life Expectancy" from Country;

1.3. Selecting Rows (WHERE Clause)

We can select particular rows by using the WHERE clause which evaluates a logical expression. For example, 

SELECT Name, Continent, Region FROM Country WHERE Continent = 'Europe';

Note that 'Europe' is enclosed within a single quote as it is a value - a literal string.

We can also sort the result of the above by using the ORDER BY clause and also limit the number of rows to be displayed by the LIMIT clause viz. 

SELECT Name, Continent, Region FROM Country WHERE Continent = 'Europe' ORDER BY Name LIMIT 5;

This will show only the first 5 rows after ordering. But if we need the second 5 rows i.e. from 6 to 10, then we have to specify an offset using the OFFSET clause viz. 

SELECT Name, Continent, Region FROM Country WHERE Continent = 'Europe' ORDER BY Name LIMIT 5 OFFSET 5;

NOTE Clauses need to be in a specific order

  1. ORDER BY must be after the WHERE clause
  2. LIMIT and OFFSET needs to be at the last
  3. FROM needs to be at the first.

1.4. Selecting Columns

A particular column can be selected by specifying the column name next to the SELECT clause. The order of the columns specified in the column names list next to the SELECT clause is same as the column order retured by the query.

1.5. Counting Rows (COUNT clause)

To count the total number of rows we can do the following 

SELECT COUNT(*) FROM Country;

To count the number of filtered rows returned using a conditional, we can use the following 

SELECT COUNT(*) FROM Country WHERE Continent = 'Europe'

Even if we specify LIMIT and OFFSET in the above statement it returns the total rows matching the condition in the WHERE clause and NOT the number of rows limited by the LIMIT clause.

Also, we can specify a column name and it will return the number of rows which match the conditions and have data in them. Rows without data are excluded. We can chain conditional statements using OR, AND

1.6. Inserting Data (INSERT INTO Statement)

We can insert a row of data (not necessarily all of them filled) into the table by using the INSERT INTO statament viz 

--test.db

SELECT * FROM customer;
-- Just for knowing whether the columns are right, exists etc.

INSERT INTO customer ( name, address, city, state, zip)
VALUES ('Sherlock Holmes', '221B Baker Street', 'London', 'None', 'NW1 6XE');

INSERT INTO customer ( name, address, city, state, zip)
VALUES ('Watson', '123 Cobblestone Way', 'Bedrock', 'CA', '91234');

1.7. Updating data (UPDATE Statement)

To update an existing entry in the database we can use the UPDATE statement viz 

-- test.db

SELECT * FROM customer;
UPDATE customer SET address = '123 Music Avenue', zip = '98056' WHERE id = 5;
UPDATE customer SET address = '2603 S Washington St', zip = '98056' WHERE id = 5;
UPDATE customer SET address = NULL, zip = NULL WHERE id = 5;

NULL is a special value which just leaves the record blank

If UPDATE statement is used without a WHERE clause, then the entire table is updated.

1.8. Deleting data (DELETE FROM Statement)

We can delete a particular row from the table using the DELETE statement. 

SELECT * FROM customer WHERE id = 4;
DELETE FROM customer WHERE id = 4;
SELECT * FROM customer;
DELETE FROM customer WHERE id = 5;
SELECT * FROM customer;

2. Fundamental Concepts

2.1. Creating a table (CREATE TABLE Statement)

CREATE TABLE test (
-- Column definitions. These are also called database schema or table schema
a INTEGER,
-- Syntax : COLUMN_NAME DATA_TYPE(,) ',' is used only during separation
-- and the final column cannot have a comma
b TEXT
);

INSERT INTO test VALUES ( 1, 'a' );
INSERT INTO test VALUES ( 2, 'b' );
INSERT INTO test VALUES ( 3, 'c' );
SELECT * FROM test;

If a table already exists then it will throw an error stating the existence of a table of the same name already.

2.2. Deleting a table (DROP TABLE Statement)

CREATE TABLE test ( a TEXT, b TEXT );
INSERT INTO test VALUES ( 'one', 'two' );
SELECT * FROM test;
DROP TABLE test;
--If there is no such table, then it will throw an error
-- stating that there is no table by the given name.
-- So, to avoid this we use the following.
DROP TABLE IF EXISTS test;

2.3. Inserting into a table

[Tested using mismatching data types : It throws no error] 

CREATE TABLE test ( a INTEGER, b TEXT, c TEXT );

INSERT INTO test VALUES ( 1, 'This', 'Right here!' ); 
INSERT INTO test ( b, c ) VALUES ( 'That', 'Over there!' );  -- This will leave `a` with NULL
-- NULL is different from 0 and ''. It is a lack of value
INSERT INTO test DEFAULT VALUES;
-- This will insert a row with NULL in all  columns
INSERT INTO test ( a, b, c ) SELECT id, name, description from item;
-- We need not specify VALUES.
-- We can insert all (without using the WHERE clause) or specific records from another table.

SELECT * FROM test;

2.4. Deleting Rows (DELETE FROM Statement)

We can delete a specific (number of) row(s) by using the DELETE FROM followed by a WHERE clause which requires the condition for selecting rows for deleting. We can delete all the records of a table but NOT the table itself by just using DELETE FROM <table_name>; which will match all the rows of the table. 

SELECT * FROM test;
DELETE FROM test WHERE a = 3;
SELECT * FROM test WHERE a = 1;
DELETE FROM test WHERE a = 1;

2.5. The NULL value

We can use IS NULL and IS NULL to check if it is a value or if it lacks a value respectively. 

SELECT * FROM test;
SELECT * FROM test WHERE a = NULL;
-- This cannot be used to test if a value is null.
-- Even the previous sentence is contradictory.
-- How will we check if a value is a *lack of value*?
SELECT * FROM test WHERE a IS NULL;
SELECT * FROM test WHERE a IS NOT NULL;
INSERT INTO test ( a, b, c ) VALUES ( 0, NULL, '' );
SELECT * FROM test WHERE b IS NULL;
SELECT * FROM test WHERE b = '';
SELECT * FROM test WHERE c = '';
SELECT * FROM test WHERE c IS NULL;

DROP TABLE IF EXISTS test;
CREATE TABLE test (
a INTEGER NOT NULL,
-- This forces a and b to have a value
b TEXT NOT NULL,
c TEXT
);

INSERT INTO test VALUES ( 1, 'this', 'that' );
SELECT * FROM test;

INSERT INTO test ( b, c ) VALUES ( 'one', 'two' );
-- This will fail because it will create NULL for a but we specified a to be NOT NULL.
INSERT INTO test ( a, c ) VALUES ( 1, 'two' );
-- This will fail because it will create NULL for a but we specified a to be NOT NULL.
INSERT INTO test ( a, b ) VALUES ( 1, 'two' );  -- This will work
DROP TABLE IF EXISTS test;

2.6. Constraining Column

We can create a column with default value viz. 

DROP TABLE IF EXISTS test;
CREATE TABLE test ( a TEXT, b TEXT, c TEXT );
INSERT INTO test ( a, b ) VALUES ( 'one', 'two' );
SELECT * FROM test;

CREATE TABLE test ( a TEXT, b TEXT, c TEXT NOT NULL );
CREATE TABLE test ( a TEXT, b TEXT, c TEXT DEFAULT 'panda' );
CREATE TABLE test ( a TEXT UNIQUE, b TEXT, c TEXT DEFAULT 'panda' );
-- The UNIQUE constraint here forces the column a to be unique.
-- That is any value under the column a must not appear more than once.
CREATE TABLE test ( a TEXT UNIQUE NOT NULL, b TEXT, c TEXT DEFAULT 'panda' );

2.7. Changing a Schema (ALTER TABLE Statement)

We can change the schema by using the ALTER TABLE statement and also we can set the default value for all the rows (number of rows corresponding to the rows in the non-altered column) 

DROP TABLE IF EXISTS test;
CREATE TABLE test ( a TEXT, b TEXT, c TEXT );
INSERT INTO test VALUES ( 'one', 'two', 'three');
INSERT INTO test VALUES ( 'two', 'three', 'four');
INSERT INTO test VALUES ( 'three', 'four', 'five');
SELECT * FROM test;

ALTER TABLE test ADD d TEXT;
ALTER TABLE test ADD e TEXT DEFAULT 'panda';
-- This makes the default values of all rows under a newly created column 'panda'

DROP TABLE IF EXISTS test;

2.8. ID Columns

Method for creating an ID column is not standardized. ID columns carry the keyword PRIMARY KEY. They can be created viz. 

CREATE TABLE test (
  id INTEGER PRIMARY KEY,
  -- This PRIMARY KEY keyword is the idenitification of an ID Column
  a INTEGER,
  b TEXT
);
INSERT INTO test (a, b) VALUES ( 10, 'a' );
-- We need not give values for the ID column
INSERT INTO test (a, b) VALUES ( 11, 'b' );
INSERT INTO test (a, b) VALUES ( 12, 'c' );
SELECT * FROM test;
DROP TABLE IF EXISTS test;

PRIMARY_KEY will automatically generate a unique sequential number (forcing that column to have unique values).

2.9. Filtering data

WHERE clause is used to filter the data along with logic operators AND, OR The _ is used to match a single character. It is simlar to . in regex. The % is used to match zero or more character (anything). It is simlar to .* in regex. The IN operator is used to select from a list. 

SELECT * FROM Country;
SELECT Name, Continent, Population FROM Country 
  WHERE Population < 100000 ORDER BY Population DESC;
SELECT Name, Continent, Population FROM Country 
  WHERE Population < 100000 OR Population IS NULL ORDER BY Population DESC;
  -- By default, the NULL state is not considered.
  -- So, to include those rows with a NULL entry we use this.
SELECT Name, Continent, Population FROM Country 
  WHERE Population < 100000 AND Continent = 'Oceania' ORDER BY Population DESC;
SELECT Name, Continent, Population FROM Country 
  WHERE Name LIKE '%island%' ORDER BY Name;
  -- '%' is a wildcard operator. It is similar to .* regex.
  -- This grabs all the names which **contains** the string 'island' in them.
SELECT Name, Continent, Population FROM Country 
 WHERE Continent IN ( 'Europe', 'Asia' ) ORDER BY Name;
 -- This matches all the Country which either belongs to Asia or Europe.

2.11. Ordering Output (ORDER BY Clause)

SELECT Name FROM Country;
SELECT Name FROM Country ORDER BY Name;
SELECT Name FROM Country ORDER BY Name DESC;
SELECT Name FROM Country ORDER BY Name ASC;
SELECT Name, Continent FROM Country ORDER BY Continent, Name;
SELECT Name, Continent, Region FROM Country ORDER BY Continent DESC, Region, Name;
-- While sorting with multiple columns, the order is important.
-- The first mentioned column will be sorted first and then in the chunk that appears,
-- each group is sorted by Region and then from the chunked Region, it will be sorted by Name

2.12. Conditional Expressions

In standard SQL, 0 is considered FALSE and anything not a 0 is considered TRUE 

DROP TABLE IF EXISTS booltest;
CREATE TABLE booltest (a INTEGER, b INTEGER);
INSERT INTO booltest VALUES (1, 0);
SELECT * FROM booltest;

SELECT
    CASE WHEN a THEN 'true' ELSE 'false' END as boolA,
    -- Checking if a is not zero
    CASE WHEN b THEN 'true' ELSE 'false' END as boolB
    FROM booltest
;

SELECT
  CASE a WHEN 1 THEN 'true' ELSE 'false' END AS boolA,
  -- Checking if a is equal to a mentioned value (1)
  CASE b WHEN 1 THEN 'true' ELSE 'false' END AS boolB 
  FROM booltest
;

DROP TABLE IF EXISTS booltest;

3. Relationships

3.1. Understanding JOIN

There are several types of JOIN as discussed below

  1. INNER JOIN : It is the intersection part of two tables
  2. LEFT JOIN : It is the intersection part along with the left table entries
  3. RIGHT JOIN : It is the intersection part along with the right table entries
  4. FULL JOIN : It combines the effect of both 2 and 3

3.2. Accessing Related Tables

ON clause tells us the condition under which the two tables must join Just specifying LEFT JOIN is enough. It is by default the LEFT JOIN 

-- join example tables, left and right
CREATE TABLE left ( id INTEGER, description TEXT );
CREATE TABLE right ( id INTEGER, description TEXT );

INSERT INTO left VALUES ( 1, 'left 01' );
INSERT INTO left VALUES ( 2, 'left 02' );
INSERT INTO left VALUES ( 3, 'left 03' );
INSERT INTO left VALUES ( 4, 'left 04' );
INSERT INTO left VALUES ( 5, 'left 05' );
INSERT INTO left VALUES ( 6, 'left 06' );
INSERT INTO left VALUES ( 7, 'left 07' );
INSERT INTO left VALUES ( 8, 'left 08' );
INSERT INTO left VALUES ( 9, 'left 09' );

INSERT INTO right VALUES ( 6, 'right 06' );
INSERT INTO right VALUES ( 7, 'right 07' );
INSERT INTO right VALUES ( 8, 'right 08' );
INSERT INTO right VALUES ( 9, 'right 09' );
INSERT INTO right VALUES ( 10, 'right 10' );
INSERT INTO right VALUES ( 11, 'right 11' );
INSERT INTO right VALUES ( 11, 'right 12' );
INSERT INTO right VALUES ( 11, 'right 13' );
INSERT INTO right VALUES ( 11, 'right 14' );

SELECT * FROM left;
SELECT * FROM right;

SELECT l.description AS left, r.description AS right
  FROM left AS l
  JOIN right AS r ON l.id = r.id
  ;

-- restore database
DROP TABLE left;
DROP TABLE right;

-- sale example
SELECT * FROM sale;
SELECT * FROM item;

SELECT s.id AS sale, i.name, s.price 
  FROM sale AS s
  JOIN item AS i ON s.item_id = i.id
  ;

SELECT s.id AS sale, s.date, i.name, i.description, s.price 
  FROM sale AS s
  JOIN item AS i ON s.item_id = i.id
  ;
